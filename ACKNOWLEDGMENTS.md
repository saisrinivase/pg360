# Acknowledgments

PG360 was shaped by years of practical database diagnostics work and by seeing how valuable a portable, HTML-driven "360-degree" report can be for DBAs and consultants.

Special thanks to Carlos Sierra, Mauro Pagano, and the `eDB360` / `SQLdb360` community for the reporting approach and operator experience that influenced the direction of this PostgreSQL project.

Thank you for building and sharing tools that many practitioners have relied on for years.

## Provenance note

- PG360 is intended to contain original PostgreSQL-focused code and report logic authored for this repository.
- PG360 is conceptually similar to `eDB360` / `SQLdb360` by design because that workflow proved useful in real-world diagnostics.
- `SQLdb360` states that it is open source under `GPLv3`, so provenance must be reviewed before public release of PG360.
- If any GPL-licensed code, copied report text, CSS, templates, screenshots, or other protected material from `eDB360` / `SQLdb360` remain in this repository, either rewrite/remove that material or publish PG360 under a GPL-compatible license before release.
